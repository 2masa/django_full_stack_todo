### 3. Frontend (`ui/`) README Format

This focuses on UI features, session management, and how it connects to the `api`.

```markdown
# `ui/` (Frontend Service)

This directory contains the `todo_ui` service, a server-side web application built with **Flask**. It renders the user interface using **Jinja2** templates and **HTMX** for dynamic updates.

## üõ†Ô∏è Tech Stack

- **Flask**: Core web framework
- **Jinja2**: Server-side HTML templating
- **HTMX**: Dynamic UI updates without full page reloads
- **Requests**: HTTP client for calling the `todo_api`
- **Flask-Session**: Server-side session management
- **Redis**: Backend for Flask-Session

## ‚ú® Key Features

- **User Login:** Secure login page that obtains JWT tokens from the `api`.
- **Session Management:** Stores user JWT tokens in a server-side **Redis** session.
- **API Proxy:** All user actions (add, edit, delete) are proxied from Flask to the backend `todo_api`.
- **Dynamic UI:** Uses HTMX for modals, table refreshes, and form submissions without a single page reload.
  - `hx-post` for creating todos.
  - `hx-get` for loading edit/delete modals.
  - `hx-swap` for updating the todo list in place.

## üîë Environment Variables

This service is configured via `devops/envs/ui.env` (generated by the `devops` CLI).

| Variable | Description |
| :--- | :--- |
| `FLASK_SECURITY_KEY` | **[SECRET]** Key for signing Flask sessions. |
| `FASTAPI_BASE_URL` | Full internal URL of the `api` service (e.g., `http://172.0.0.2:7000`) |
| `FLASK_PORT` | Port to run Flask on (e.g., `5000`) |
| `FLASK_DEBUG` | Enable Flask debug mode (e.g., `true`) |
| `REDIS_HOST` | Redis service IP (in Docker) |
| `REDIS_PORT` | Redis service port |

## üèÉ Running

This service is not intended to be run standalone. It must be started via the main `devops` CLI.

# From the root /devops/ directory
uv run cli service start-dev

```
